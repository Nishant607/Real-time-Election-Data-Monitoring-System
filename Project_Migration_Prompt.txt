Provide a complete, step-by-step implementation guide to build a "Real-time Election Data Monitoring System" from scratch. The system must have a robust workflow, a beautiful modern GUI, and a backend powered by MySQL.

### **Tech Stack Requirements:**
1. **Frontend:** React.js (via Vite), HTML, CSS, JavaScript, Tailwind CSS (for styling), Framer Motion (for animations), and Lucide React (for icons).
2. **Backend:** Python, Django, Django REST Framework, and Django Channels (for real-time WebSockets).
3. **Database:** MySQL (Django configured with `mysqlclient`).

---

### **1. Database Schema & Django Models (MySQL)**
You will use MySQL as the primary relational database. Implement the following models logically linked via Foreign Keys:

- **User Model (Custom):** Must have fields for email, username, and `role` (Admin vs Standard User).
- **Election:** `name` (CharField), `start_date` (DateField), `end_date` (DateField). Add a property/method to determine status (Upcoming, Active, Completed).
- **Candidate:** `election` (ForeignKey), `name` (CharField), `party` (CharField).
- **VoteRecord:** `candidate` (ForeignKey), `votes` (IntegerField), `timestamp` (DateTimeField), `status` (Choices: Pending, Approved, Rejected).
- **Anomaly:** `candidate` (ForeignKey), `previous_votes`, `current_votes`, `detected_at` (DateTimeField), `reason` (CharField), `vote_record` (OneToOneField).
- **Alert:** `anomaly` (ForeignKey), `message` (CharField), `severity` (CharField), `status` (Active, Investigating, Resolved, etc.), and investigation fields (notes, action taken).
- **ActivityLog:** `action` (CharField), `created_at` (DateTimeField).

**Critical Business Logic on the Backend:**
When a `VoteRecord` is being saved, compute the difference in votes compared to the last record for that candidate. If the vote numbers jump by >= 50% or if updates happen faster than 60 seconds, it is considered an anomaly. The system must:
1. Mark the new vote as 'Pending'.
2. Create an `Anomaly` object.
3. Generate a High-Severity `Alert`.
4. Trigger an Email Notification to admins using Django's `send_mail`.
5. Log this into the `ActivityLog`.

---

### **2. Backend API & Real-time Workflow (Django REST)**
- Build modular REST API endpoints for fetching elections, adding candidates, submitting votes, listing alerts, and updating alert statuses.
- Configure `djangorestframework-simplejwt` for secure Authentication.
- Use **Django Channels** and a Redis backing layer to implement WebSockets. Whenever a new vote or alert is generated via the API, broadcast the payload over WebSocket channels.
- Configure CORS to allow the React frontend to communicate with the Django application.
- Supply proper setup commands for the `DATABASES` settings dict inside `settings.py` for MySQL optimization.

---

### **3. Frontend Architecture (React + Vite)**
Build a highly scalable and interactive React project. 

**Routing Structure (react-router-dom):**
- `/` - Landing Page (Immersive, Glassmorphism, animations). Acts as a stunning gatekeeper.
- `/login` - Credential entry and authentication context layer.
- `/dashboard` - Protected. High-level metric overviews.
- `/elections` - Protected. View all current elections.
- `/candidates` - Protected. Monitor candidate standing.
- `/alerts` - Protected. Live-updating queue of suspicious vote spikes.
- `/alerts/investigate/:id` - Admin Protected. Deep dive into a flagged anomaly.
- `/analytics` - Protected. Graphical charts (Recharts/Chart.js) showing vote trends.
- `/reports` - Protected. Generate exportable reports.
- `/create-account` - Admin Protected. Form to provision new node operators.
- `/audit-logs` - Admin Protected. Detailed chronological system events.

**Design & UI Rules:**
- The Application should be wrapped in a visually striking Dark Mode theme (`bg-[#020617]` layout) with glowing gradients, blurs, and glassmorphic panels.
- Add a Persistent Sidebar Navigation for logged-in users.
- Role-based Access Control (RBAC): Admin links (Create Account, Audit Logs) must be strictly hidden from normal 'User' roles. Route guards must redirect unauthorized visitors.
- Connect to Django Channels via a WebSocket provider in the React layer so that Dashboards, Analytics, and Alerts automatically reflect state without requiring browser refreshes.
